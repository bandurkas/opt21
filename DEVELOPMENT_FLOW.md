# Development Workflow (Флоу разработки)

Наш стандартный процесс внесения любых изменений в проект:

1. **Архитектура:** Пишем план (Implementation Plan). Обдумываем "что" и "зачем".
2. **Код:** Пишем сам код согласно утвержденному плану.
3. **Ревью:** Обязательный Code-Review написанного кода перед тестами.
4. **Тест:** 
   - Если это поиск преимущества (edge) → Backtest + Holdout (OOS).
   - Если это исправление ошибки (bug) → Пишем/прогоняем Unit-тесты или любые другие тесты.
5. **Ревью результатов:** Еще раз смотрим глазами на логи и результаты тестов.
6. **Деплой:** `commit` → `push` → `git pull` на VPS сервере.
7. **Верификация:** Проверяем, что контейнер `Up`, в логах `0 ошибок`, и гейт (наша бизнес-логика) цел.
8. **Синк:** Убеждаемся, что код идентичен на (local = GitHub = VPS), обновляем документацию и рабочую память бота.
